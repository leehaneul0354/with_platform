# WITH Platform — Architecture Decision Records (ADR)

이 폴더에는 **라이브러리 선택**, **아키텍처 결정**, **주요 기술 선택**에 대한 **이유(Why)**를 기록합니다.

## 기록 시점

- 새 패키지/라이브러리 도입 시
- 상태 관리·라우팅·API 레이어 등 구조 결정 시
- Mobile/Web 분기 전략 등 플랫폼별 결정 시

## 파일 규칙

- 파일명: `YYYY-MM-DD-짧은-제목.md` (예: `2025-02-14-state-management-choice.md`)
- 각 문서에 **결정 내용**과 **선택 이유(Why)**를 명시

## 템플릿

새 의사결정 문서 작성 시 아래 구조를 사용할 수 있습니다.

```markdown
# [제목]

- **날짜**: YYYY-MM-DD
- **상태**: 제안 / 채택 / 폐기

## Context
어떤 문제/요구사항 때문에 결정이 필요했는지.

## Decision
무엇을 선택했는지 (라이브러리, 패턴, 아키텍처 등).

## Why (Reasoning)
왜 그 선택을 했는지. 대안과 비교한 이유를 간단히.
```
